## What This Is

The Scarcity–Denial Framework is a structured analytical architecture 
for diagnosing why systems fail to protect their binding scarce 
resources. It comprises a four-sentence Canon and three sequential 
derived instruments. Applied together, the instruments identify the 
binding scarce resource governing system stability, map the denial 
architecture against that resource, and detect downstream migration 
of the denial boundary.

The framework is designed to produce findings that are traceable, 
reproducible, and institutionally communicable — and to distinguish 
between denial architectures that are absent, mislocated, or drifting. 
This distinction drives the minimum intervention logic: what needs to 
be built, relocated, or strengthened, and where.

## How to Use the Framework

The instruments run sequentially: SRIT → DAI → DDA.

**Start here:** Read `SDF_Runtime_Guide` before beginning any run.
The Runtime Guide sequences the complete three-module pipeline from 
first session to final artefact — what to open, what to upload, what 
to carry between sessions, and what decisions to make at each module 
boundary.

The framework runs on two platforms operating in parallel:
- **Claude Sonnet 4.6** — companion session for system description
  preparation, active commentary, boundary checks, and Run Record
  formatting
- **ChatGPT GPT-5.4 Thinking Extended** — analytical execution across
  three separate module sessions

The instrument prompt files are not operated directly by the user.
They are loaded into the execution sessions as governed by the
Runtime Guide. The `SDF_User_Manual` is the reference document for
understanding what each instrument produces and how to interpret
outputs.

No modification of the instruments is required or permitted for any 
legitimate use. Specialisation to any system, sector, or context is 
achieved entirely through the System Definition Statement — constructed 
in the Claude companion session before SRIT begins.
